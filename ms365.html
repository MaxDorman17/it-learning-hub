MS365 Admin

The Microsoft 365 admin work behind most tickets. Learn the map of admin centers first — knowing which portal does what saves you more time than anything else.

🗺️The admin centers — which portal does what▾

Microsoft 365 isn't one console — it's several, each owning part of the estate. Getting lost here is the #1 time-sink for new engineers, so learn this map.

  • Microsoft 365 admin center — admin.microsoft.com — users, licences, groups, service health, billing. Your home base.
  • Entra admin center — entra.microsoft.com — identity: sign-in logs, MFA, Conditional Access, SSPR.
  • Exchange admin center — admin.exchange.microsoft.com — mailboxes, shared mailboxes, mail flow, message trace.
  • Teams admin center — admin.teams.microsoft.com — Teams policies, phone/calling, provisioning.
  • SharePoint admin center — [tenant]-admin.sharepoint.com — sites, sharing, storage.
  • Defender / Security — security.microsoft.com — quarantine, threats, alerts.
  • Intune — intune.microsoft.com — device and app management.
Tip: most tickets touch one or two of these. Ask yourself "is this about who they are (Entra), their email (Exchange), their files (SharePoint), or their device (Intune)?" and go straight there.
👤Users & licensing▾

Everything about a person's account lives under Users → Active users in the 365 admin center.

Creating & managing users

  • Follow the naming convention (usually firstname.lastname@domain).
  • Set a usage location — required before some licences will assign.
  • Use Block sign-in to instantly cut access without deleting the account (leavers).

Licensing

Licences come from a pool the tenant has bought. You assign a free one — you don't buy per user unless the pool is empty (which needs approval).

Least privilege: most users need no admin role. Only assign roles (like Helpdesk Admin) with sign-off — over-granting is a security risk.
📧Exchange Online▾

Email lives in the Exchange admin center. The things you'll touch most:

  • Mailboxes — a user's own email. Shared mailboxes (e.g. info@) need no licence under 50 GB.
  • Permissions — Full Access (read it), Send As (send as it), Send on Behalf.
  • Distribution lists — email one address, everyone in it receives it.
  • Message trace — prove what happened to an email (delivered / blocked / quarantined). Your evidence tool for "I never got it".
  • Quarantine — release legitimate mail wrongly caught as spam (from the Security portal).
See also: the Shared mailbox and Outlook runbooks put this into practice.
🗂️SharePoint & OneDrive▾

SharePoint hosts team sites, document libraries and intranets; OneDrive is each person's personal file store. Both are managed in the SharePoint admin center.

Permissions — the thing that generates tickets

  • Owner — full control, can manage permissions.
  • Member — can edit content.
  • Visitor — read-only.

"Access denied" is almost always a permissions problem, not a technical fault — check the user's level and the sharing link before anything else.

Recovery: deleted sites are recoverable for 93 days; deleted files go to the recycle bin (site, then second-stage). You can often recover before escalating.
🛡️Entra ID & security basics▾

Identity is managed in the Entra admin center. As L1 you'll mostly read here and make small, safe changes.

  • Sign-in logs — the truth about why a login failed, and from where. Check these before guessing.
  • Authentication methods — where you reset/re-register a user's MFA.
  • Groups — security groups (grant access) vs Microsoft 365 groups (team + mailbox + site).
  • Conditional Access — policies that may block a sign-in for a reason (location, device). Recognise it; don't change it without sign-off.
Security instinct: unexpected sign-ins from odd countries, or a user suddenly with mailbox forwarding rules, means possible compromise — escalate to security, don't sit on it.
💻PowerShell for M365▾

The portals cover 90% of L1 work, but PowerShell is faster for bulk changes and some settings only exist there. You'll grow into it — start by recognising it, then using copy-paste commands.

  • Exchange Online — Connect-ExchangeOnline then cmdlets like Get-Mailbox.
  • Microsoft Graph — Connect-MgGraph for users, groups, licences (the modern module).
Ready-made commands: the Cheat Sheets page has copy-paste Exchange and Graph one-liners for the common tasks.
