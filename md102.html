🧠Key concepts & exam traps▾

Compliance vs Conditional Access (the #1 trap)

A compliance policy only labels a device Compliant / Not compliant — it blocks nothing on its own. Conditional Access is what enforces "only compliant devices get email/files". They're a pair. Compliance is evaluated at check-in, not instantly.

Join types

  • Microsoft Entra joined — corporate-owned, cloud-managed. The default for company devices.
  • Entra registered — personal / BYOD; the device is known but the user keeps ownership.
  • Hybrid Entra joined — joined to on-prem AD and Entra (orgs with existing AD).

Enrollment

Automatic enrollment = Entra → Mobility (MDM) → Intune, MDM user scope = All. Needs an Intune licence. Windows Autopilot pre-registers a device (hardware hash) so it self-configures at OOBE; the Enrollment Status Page (ESP) can block the desktop until required apps/policies land.

Remote actions — know the difference

  • Wipe — factory reset, removes everything. Lost/stolen or full re-provision.
  • Retire — removes only company data, leaves personal. The BYOD choice.
  • Fresh Start — removes apps, keeps the device managed (fix a messy Windows).

Other must-knows

  • BitLocker recovery keys are escrowed to Entra/Intune — look up by Key ID; you can rotate them.
  • The Mark device noncompliant action is automatic, mandatory, defaults to 0 days.
  • Apps: Required auto-installs; Available is opt-in via Company Portal. Win32 apps need correct detection rules + dependencies.
🧪Lab 01 — Create a Windows compliance policy▾

Domain 1 + the Implement device compliance ↗ module. Do it for real in the Intune admin center ↗ (intune.microsoft.com) in the test tenant — don't just read it. Full docs: Create a compliance policy ↗.

A · Create the policy

  1. Devices → Compliance → Policies → Create policy.
  2. Platform: Windows 10 and later → Create.
  3. Basics: Name Windows – Baseline Compliance (LAB).
  4. Compliance settings: turn on Require BitLocker, Secure Boot, require a password (min length 6), encryption.
  5. Actions for noncompliance: note the auto-added Mark device noncompliant (0 days).
  6. Assignments: assign to a test group (user or device group).
  7. Review + create. Confirm it shows Assigned = Yes.

B · The tenant trap

Check Devices → Compliance → Compliance settings → "Mark devices with no compliance policy as". If it's Not compliant + Conditional Access is on, devices with no policy get blocked. Classic ticket.

C · Verify

On a device: Settings → Accounts → Access work or school → Sync, then check Devices → the device → Device compliance for pass/fail per rule.

Stretch: add a Conditional Access policy that requires a compliant device, so you see it actually enforce — but only against a test user, with a break-glass admin excluded.
